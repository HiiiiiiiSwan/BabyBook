# 生成中页面（GeneratingView）逻辑小结

## 一、页面状态

| 状态 | 含义 |
|------|------|
| RUNNING / PENDING | 后端正在生图，进度 < 90% |
| COMPLETED | 后端生图完成，进入 90%–100% 预加载图片阶段 |
| FAILED | 后端生图失败 |
| 轮询超时 | 5 分钟内任务未完成 |

---

## 二、进度 < 90%：生图阶段

- 每 **3 秒** 轮询一次后端任务状态
- 单次请求超时 **30 秒**
- 连续轮询失败：
  - 第 1 次：显示「网络开小差了，正在重试…」
  - 第 3 次及以上：显示「网络连接异常，请检查网络后重试」
- **不会因为连续失败就跳转失败页**
- 只有总轮询时间达到 **5 分钟** 后，才判定超时并跳转失败结果页

---

## 三、进度 90%–100%：预加载图片阶段

进入条件：后端返回 `COMPLETED`，开始下载并预加载绘本图片。

### 首次流程

1. `saveGeneratedBook()` 下载图片并保存到本地
2. `preloadImage()` 把图片加载到内存
3. 成功后跳转完成页

### 任一环节失败

进入 `retryDownloadAndPreload()` 刷新重试：

| 项目 | 当前逻辑 |
|------|---------|
| 最大重试次数 | **5 次** |
| 重试间隔 | 2 秒 / 4 秒 / 6 秒 / 8 秒，逐步递增 |
| 提示文案 | 「网络异常，正在刷新重试，请稍后」 |
| 提示出现时机 | 进入重试函数时显示 |
| 提示消失时机 | **每次开始新一轮重试时先隐藏**；若后续又失败再重新显示 |

### 网络恢复后

- 新一轮重试开始时会先隐藏 Toast
- 下载成功 → 继续预加载 → 跳转完成页
- 全程不再让用户看到过期的「网络异常」提示

---

## 四、跳转完成页前的等待

`navigateToCompleteWithMinDelay()`：

- 最多等待 **5 秒** 让图片预加载完成
- 5 秒内完成：再展示至少 **0.8 秒**「生成完成！」后跳转
- 5 秒仍未完成：触发 `retryDownloadAndPreload()` 兜底重试

---

## 五、跳转失败页的场景

| 场景 | 来源 |
|------|------|
| 后端任务状态为 `FAILED` | `checkTaskStatus()` |
| 轮询总时长超过 5 分钟 | `handleTimeout()` |
| 预加载图片 5 次重试全部失败 | `retryDownloadAndPreload()` |
| 恢复已失败订单时获取任务信息失败 | `handleFailedEntry()` |

---

## 六、核心改动

本次对话主要调整了预加载失败后的重试和提示策略：

- 重试次数从 3 次提升到 **5 次**
- 重试间隔从固定 2 秒改为 **阶梯递增**
- Toast 提示从「一直显示到跳转」改为「**每次重试开始时先隐藏，失败再显示**」
