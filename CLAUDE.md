# BabyBook（宝贝绘本）项目规范

## 项目定位

面向 1~3 岁儿童家庭的 AI 个性化绘本定制 iOS App。用户上传 1 张宝宝照片，1 分钟内生成宝宝为主角的专属绘本。

## 技术栈

- **App**: SwiftUI + StoreKit2 + PDFKit + PhotosPicker
- **Backend**: Node.js + NestJS
- **Database**: PostgreSQL (Supabase Free)
- **AI**: 豆包 Seedream 5.0 Lite（通过后端调用，禁止客户端直接访问）
- **部署**: 后端需自行部署，MVP 不使用对象存储

## 关键业务规则（必须遵守）

### 1. 用户体系
- **无需登录/注册/手机号/Apple登录**
- 采用匿名设备模式，首次启动生成 `device_id` 存入 iOS Keychain
- 用于订单关联和生成记录关联

### 2. 支付流程（关键！）
- 仅支持 Apple IAP，商品类型为 Consumable
- **支付成功后必须由后端自动创建生成任务**，禁止客户端直接调用生成接口
- 原因：用户可能在支付后关闭 App、崩溃、断网、锁屏，这些都不能影响生成流程
- 客户端通过轮询订单状态恢复任务进度

### 3. 订单状态流转
```
UNPAID → PAID → GENERATING → SUCCESS / FAILED
```
- 失败自动重试 2 次，仍失败则标记 FAILED，由客服人工处理
- 退款状态：REFUND

### 4. AI 生成策略（成本控制优先）
- **单本绘本仅调用一次生图模型**，输出单张 2048×2048 九宫格图片
- 禁止逐页生成、多轮生成、复杂角色训练
- 输入：空白模板图 + 宝宝照片（多图参考模式）
- 输出：包含 9 页绘本内容的完整九宫格图片
- 后续扩展：V2 单页独立生成，V3 多页连续绘本

### 5. 数据隐私（数据最小化原则）
- **宝宝照片**：上传后仅用于 AI 生成，生成完成立即删除，不保存
- **绘本图片**：生成后立即删除，不保存
- **PDF 文件**：仅保存在用户本地 iOS Documents 目录，不上传云端
- **保留数据**：仅保存订单记录（order_id, device_id, book_id, payment_id, amount, status, 时间戳）
- 用户首次下载时提示：绘本仅保存在当前设备，卸载后无法恢复

### 6. API 安全
- API Key 仅存储在后端，禁止写入客户端
- 客户端不得直接访问 AI 模型
- 所有 AI 调用必须经过后端

## 设计规范（必须遵守）

### 色彩系统
- 主色（CTA/按钮/价格）：`#F28C28` Baby Orange
- 背景色：`#FFF9F2`
- 页面背景：`#FFFDF9`
- 成功色：`#8BC34A`
- 主文字：`#222222`
- 辅助文字：`#666666`
- 三级文字：`#999999`
- 边框：`#F0E8DE`
- 分割线：`#F5F0E8`

### 字体规范
- 中文：PingFang SC
- 英文：SF Pro Display
- Fallback：SF Pro Display, PingFang SC, sans-serif

### 圆角规范
- 卡片：24px
- 绘本封面：28px
- 按钮：999px（全圆角）
- 弹窗：32px

### 阴影规范
- 默认卡片：`0 8px 30px rgba(0,0,0,0.06)`
- 悬浮卡片：`0 12px 40px rgba(0,0,0,0.08)`
- 原则：像云朵漂浮感，禁止重阴影

### 间距规范
- 统一 8pt Grid：4, 8, 12, 16, 24, 32, 48, 64
- 页面边距：24px
- 模块间距：24px
- 卡片内边距：16px

### 按钮规范
- 主按钮高度：56px，背景 `#F28C28`，文字白色，全圆角
- 次按钮：白色背景 + `#F28C28` 边框
- 禁用按钮：背景 `#E5E5E5`，文字 `#999999`

### 插画风格
- Warm Storybook 3D（暖色系绘本收藏风）
- 3D 儿童绘本风，Q 版比例，圆润造型，柔和光影，低复杂度细节
- 常用元素：云朵、草地、热气球、小熊、小兔子、星星、书本、树叶
- **禁止**：写实风、科技风、二次元风、扁平互联网插画

### Vibe Coding 铁律
1. 所有页面背景使用 `#FFF9F2`
2. 所有主按钮使用 `#F28C28`
3. 大量留白
4. 优先圆角卡片
5. 优先插画而非图标
6. 不使用科技蓝
7. 不使用渐变紫
8. 不使用深色模式
9. 不使用复杂导航
10. 不出现 AI 工具感设计

### 设计成功标准
- ❌ 像 AI 工具 / 教育 App / 儿童游戏 / 互联网运营产品
- ✅ 像一本值得收藏十年的宝宝成长纪念绘本

## 页面清单（8 个页面）

1. **首页**：绘本列表展示（封面、名称、价格），支持查看详情和一键定制
2. **绘本详情**：预览图、页数、内容介绍，一键定制按钮
3. **上传照片**：相册选择或拍照，限制 1 张，最大 10MB，必须检测到人脸
4. **支付页**：绘本名称 + 价格，Apple IAP 支付
5. **支付成功**：生成任务创建成功，展示生成中动画（预计 30~60 秒）
6. **AI 生成中**：进度动画 + 文案"正在为宝宝制作专属绘本"，支持取消生成
7. **绘本完成**：展示封面，支持下载 PDF 电子版和获取实体书，提示及时保存
8. **我的绘本**：本地已下载 PDF 列表（iOS Documents），支持查看、删除、分享

## 页面导航说明

- **首页 → 绘本详情**：点击绘本封面或卡片
- **绘本详情 → 上传照片**：点击「一键定制」按钮，弹出 `UploadPhotoSheet` 底部弹窗
- **首页 → 上传照片**：点击底部「一键定制」按钮，同样弹出 `UploadPhotoSheet`（共用组件）
- **上传照片 → 支付页**：选择照片并创建订单后，自动跳转
- **支付页 → 生成中页**：支付成功后跳转
- **生成中页 → 绘本完成页**：生成完成后自动跳转（模拟器 5 秒）
- **绘本完成页 → 首页**：点击左上角返回按钮（系统导航返回）

## 关键组件说明

### UploadPhotoSheet（共用弹窗）
- **定义位置**：`HomeView.swift` 中定义，首页和绘本详情页共用
- **触发方式**：`showUploadSheet = true` 通过 `.overlay` 条件渲染
- **注意**：首页 `.overlay` 条件中不要加 `let book = selectedBook` 的额外解包判断，否则可能导致弹窗无法显示

### 导航标题规范
- 使用 `.toolbar { ToolbarItem(placement: .principal) { ... } }` 统一设置
- 字号：18pt，字重：bold，颜色：`#222222`
- 示例页面：绘本详情页、生成中页

## 后端 API 清单

| 接口 | 方法 | 路径 | 说明 |
|------|------|------|------|
| 创建订单 | POST | `/api/order/create` | 参数：book_id, device_id |
| 支付验证 | POST | `/api/payment/verify` | 参数：order_id, receipt；成功后服务端自动触发生成任务 |
| 查询任务状态 | GET | `/api/task/{id}` | 返回：GENERATING / SUCCESS / FAILED |
| 下载 PDF | GET | `/api/book/{orderId}/pdf` | 返回 pdf_file |

## 异常处理

- AI 失败：自动重试 2 次
- 支付成功但生成失败：订单保持 FAILED，后台可查询，客服人工处理
- 用户关闭 App：重新打开后继续轮询订单状态
- 网络中断：自动恢复轮询

## MVP 成功指标（上线后 30 天）

- 支付转化率 ≥ 5%
- 绘本生成成功率 ≥ 95%
- 退款率 ≤ 3%
- 观察指标：实体书点击率、用户评价

## 开发注意事项

1. **SwiftUI 推荐组件**：NavigationStack, ScrollView, LazyVGrid, AsyncImage, PhotosPicker, PDFKit, StoreKit2
2. **Design Token 必须统一**：禁止硬编码颜色和间距，所有页面引用统一 Token
3. **文件命名规则**：`book_{orderId}.png` / `book_{orderId}.pdf`
4. **生成超时**：AI 调用超时设置为 300 秒（九宫格生成较慢）
5. **PDF 导出**：AI 生成完成后自动导出 PDF，用于用户下载、分享、实体书打印
6. **图片资源路径**：模板图片使用绝对路径（开发环境），生产环境需改为 Bundle 资源方式
