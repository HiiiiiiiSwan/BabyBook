# 生成中页面（GeneratingView）逻辑小结

## 一、页面入口

生成中页面有 3 种进入方式：

| 入口 | 来源 | 进入时状态 |
|------|------|------------|
| 支付成功后正常进入 | `PaymentView` / `UploadPhotoView` / `UploadPhotoSheet` | `order.status = GENERATING` |
| 恢复已支付订单 | `HomeView` 恢复弹窗 | `order.status = PAID` |
| 恢复已成功/失败订单 | `HomeView` 恢复弹窗 | `order.status = SUCCESS / FAILED` |

---

## 二、核心状态流转

```
进入 GeneratingView
    ↓
持久化订单为 GENERATING 状态（用于杀端/崩溃后恢复）
    ↓
开始生成：
  ├─ 模拟器：5 秒后模拟完成
  └─ 真机：轮询后端任务状态
    ↓
任务状态 COMPLETED
    ↓
下载九宫格图片 → 保存本地 → 预加载到内存
    ↓
跳转 CompleteView（完成页）
```

---

## 三、轮询阶段网络异常处理

在 AI 真正生成图片期间（任务状态为 `PENDING` / `RUNNING`），页面每 3 秒轮询一次任务状态：

| 网络情况 | 反馈 |
|----------|------|
| 单次轮询失败 | 底部非阻断 Toast：**"网络开小差了，正在重试..."** |
| 连续 3 次失败 | 底部非阻断 Toast：**"网络连接异常，请检查网络后重试"** |
| 网络恢复、轮询成功 | Toast 自动消失，继续正常进度 |
| 持续异常直到 5 分钟超时 | 跳转 `FailureResultView`（失败结果页，联系客服） |

---

## 四、90%~99% 阶段下载/预加载失败处理

后端任务状态变成 `COMPLETED` 后，进入 90%~99% 缓冲阶段，此时做三件事：

1. 从 `task.resultUrl` 下载九宫格图片
2. 保存到本地 Documents 目录
3. 预加载到内存

### 失败处理

| 失败点 | 行为 |
|--------|------|
| 下载失败 | 不跳转完成页，停留在生成中页面 |
| 预加载失败 | 不跳转完成页，停留在生成中页面 |
| 5 秒等待超时仍未加载完成 | 触发刷新重试机制 |

### 刷新重试机制

- 显示 Toast：**"网络异常，正在刷新重试，请稍后"**
- 最多重试 **3 次**
- 每次间隔 **2 秒**
- 仅重新从 `resultUrl` 下载已生成的图片
- **不会重新调用 AI 生图，不会产生额外 API 费用**
- 3 次都失败 → 跳转 `FailureResultView`（失败结果页）

---

## 五、失败结果页

生成中页面可能跳转失败页的情况：

| 触发条件 | 说明 |
|----------|------|
| 后端任务状态为 `FAILED` | AI 生成失败 |
| 前端轮询超时 | 5 分钟仍未收到终态 |
| 90%~99% 下载/预加载 3 次失败 | 网络异常无法获取已生成图片 |
| 支付成功后后端验证网络失败 | Apple 已扣款但后端异常 |

失败页展示：

- 失败图标 + "绘本生成失败"
- 失败原因文案
- 客服二维码
- "返回首页"按钮

---

## 六、关键设计原则

1. **不误导用户**：只有 Apple 支付真正完成扣款后，才进入生成流程；支付失败只弹窗提示，不跳转。
2. **已扣款必进客服页**：只要 Apple 已扣款但后端异常，就进入失败结果页，避免用户反复尝试导致困惑。
3. **不重复收费**：90%~99% 阶段的重试只是重新下载已生成图片，不会再次调用 AI。
4. **杀端可恢复**：生成中页面进入时会持久化订单状态，App 重启后通过 `restoreOrderIfNeeded` 恢复。
5. **立即 finish StoreKit 交易**：支付成功后立即 `transaction.finish()`，防止 StoreKit 复用交易导致新订单跳过支付。

## 七、涉及修改的文件

| 文件 | 主要改动 |
|------|----------|
| `GeneratingView.swift` | 新增 Toast、刷新重试、网络异常监听 |
| `AppServices.swift` | 新增 `pollingFailureCount` 计数 |
| `PaymentService.swift` | 新增 `paidButServerError` 错误类型，提前 `finish()` 交易 |
| `PaymentView.swift` | 已扣款后进失败结果页 |
| `UploadPhotoView.swift` | 已扣款后进失败结果页 |
| `HomeView.swift` 的 `UploadPhotoSheet` | 已扣款后进失败结果页 |
| `FailureResultView.swift` | 调整 UI，修复返回首页按钮 |
